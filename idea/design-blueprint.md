# 精神避难所 v5 — 设计蓝图

> 最后更新：2026-05-15
> 域名：hzd-ms.com

---

## 一、项目定位

一个让你忘记"这是网页"的数字精神避难所。

核心理念：
- **不是给用户更多东西，而是做得更少、更精准**
- **身体感**——入场→坐→躺→闭眼→对话，每个动作对应一种心理状态转变
- **隐藏而非展示**——深度来自偶然发现，不是显式告知
- **真正的隐私**——AI 对话不留痕、不出声、不存储

---

## 二、入场仪式

**眼镜入场动画（~3-4秒）**

画面：纯黑背景 → 中央浮现一副自然风格的眼镜 → 镜片内部透出场景画面 → 眼镜由远及近向用户脸部拉近 → 当眼镜"覆盖"到用户眼前时，过渡到第一人称场景视角。

眼镜风格：偏向自然/文艺感，非科技 VR 头显。传递"换个视角看世界"的意味，而非"进入虚拟空间"。

技术：Three.js 构建眼镜 3D 模型 + CSS/WebGL 过渡动画。

## 三、场景系统（共 2 个场景，每分钟自动轮换）

### 3.1 海崖场景

- **视角**：用户坐在海边悬崖上，面朝大海方向
- **受限视角**：左右各旋转 ±75°，模拟人颈自然转动范围
- **画面要素**：
  - 前方：大海、波浪拍岸、海鸥偶尔掠过
  - 左侧：连着的崖壁延伸而去
  - 左侧尽头：海上的一座孤岛
  - 整体是崖壁内角，非完全开阔海面——遮蔽带来私密感和真实感
- **固定机位拍摄的广角视频**：背景层

### 3.2 森林场景

- **视角**：用户坐姿，身处于森林中
- **受限视角**：左右 ±75°
- **画面要素**：
  - 周围树木、灌木
  - 树冠间隙的蓝天可见
  - 偶尔树叶飘落、光斑移动

### 3.3 自动轮换

- 默认每个场景 60 秒自动切换
- 双击后锁定当前场景，停止自动轮换

## 四、交互分层

### 4.1 自由凝视（无操作）
- 鼠标/陀螺仪控制视角，仅限 ±75° 范围
- 场景持续"呼吸"：波浪、树叶、海鸥、光斑等

### 4.2 凝视彩蛋（鼠标滞留 3 秒）
- 鼠标在某处停留超过 3 秒 → 隐藏彩蛋触发
- 具体彩蛋内容后续补充
- 无任何 UI 提示，用户偶然发现

### 4.3 双击：身体姿态转换

**坐→躺**：镜头从面向前方平滑过渡到仰天视角

| 海崖躺下后 | 森林躺下后 |
|-----------|-----------|
| 开阔蓝天白云 | 树木向上延伸 |
| 风软软的声音 | 屏幕左右边界有较高的草 |
| | 正中间是树冠间隙露出的蓝天（约占屏幕 1/8）|

**再双击→坐**：镜头回到坐立视角

### 4.4 三击："闭眼"

- 屏幕变朦胧模糊（非全黑——模拟光线透过眼皮的效果）
- 环境声音增大
- 平时被掩盖的细微声音浮现（风穿过树叶、远处鸟鸣等）
- 再三击→回到清晰

### 4.5 长按（或特殊手势）：进入无痕对话

详见第五部分。

## 五、"黑屏聊天"——无痕情感对话

### 5.1 进入方式
在"闭眼"模糊状态下，长按或特定手势进入。

### 5.2 体验设计
- 屏幕近乎全黑
- 一位 agent 从黑暗中浮出一条问候语
- 问候语示例："有什么想和我说的吗？"
- 文字浮现后缓慢淡出，不留下痕迹
- 用户回复后，自己的文字也浮现→消失
- 整个界面**只有光标**，没有对话框、没有聊天记录、没有头像
- 像在黑暗中对自己说话

### 5.3 Agent 设计
- **人格**：温暖、安静、不带评判的倾听者
- **能力**：拥有长期记忆和短期记忆，记住用户是谁、说过什么
- **用户数据库**：本地存储（IndexedDB），用于维护用户状态和记忆
- **完全隐私**：数据只存在于用户本地设备

### 5.4 技术路线

**主力方案：WebLLM（浏览器内本地模型）**
- 模型在用户设备本地运行，不经过任何服务器
- 零 API 成本，真正买断制
- 候选模型：Qwen 2.5 7B（中文能力强）
- 首次加载需下载模型文件（4-8GB），需 WebGPU 支持
- 数据存本地 IndexedDB

**兜底方案：Cloud API 代理**
- 用户设备不支持 WebLLM 时自动降级
- 通过 Vercel Serverless Functions 代理 API 调用
- 消耗预设额度（买断价内含额度池）

## 六、架构设计

### 6.1 网站整体架构

```
入场仪式（Three.js 眼镜动画，~3-4s）
    ↓
进入 3D 场景（默认海崖，颈动视角 ±75°）
    ├── 60s 自动轮换：海崖 ↔ 森林
    ├── 双击 → 锁定场景 + 躺姿
    │    ├── 再双击 → 坐姿
    │    └── 三击 → "闭眼"模糊模式
    │         ├── 再三击 → 恢复清晰
    │         └── 长按 → 进入无痕对话
    └── 凝视 3s → 隐藏彩蛋
```

### 6.2 技术栈

| 层 | 技术 |
|----|------|
| 3D 场景/视角 | Three.js |
| 视频图层 | HTML5 Video + WebGL 纹理映射 |
| 音频空间化 | Web Audio API |
| 本地 AI 对话 | WebLLM（Qwen 2.5 或 DeepSeek） |
| Cloud 兜底 | Vercel Functions + Claude API |
| 本地存储 | IndexedDB（用户记忆 + 访问状态）|
| 部署 | GitHub Pages（静态）+ Vercel（Serverless Functions）|

### 6.3 项目结构（预期）

```
/
├── index.html          # 入口
├── entrance/           # 入场仪式模块
├── scenes/             # 场景渲染引擎
│   ├── sea-cliff/      # 海崖
│   └── forest/         # 森林
├── audio/              # 音频引擎
├── chat/               # 无痕对话系统
├── assets/             # 静态资源
│   ├── video/          # 实景视频素材

```

## 八、商业模式

### 定位
买断制：199 元，永久使用。

### 理由
- 网站的核心资产是"体验"而非"服务"
- 场景和交互是静态的，无边际成本
- 买断制降低消费心理门槛
- 和月租制产品（充钱才能用的 AI 聊天）形成差异

### AI 对话成本处理

**方案：WebLLM 本地 + Cloud 额度兜底**

```
199 元买断包含：
├── 沉浸场景体验（永久，零边际成本）
├── 本地 AI 对话（永久，WebLLM 在用户设备上跑）
│   └── 用户记忆存本地 IndexedDB
└── Cloud API 额度 300 次（兜底，用完可加购额度包）
    └── 加购：29 元 = 200 次对话
```

- 大部分 PC 用户通过 WebLLM 零成本维持
- 手机端/不支持 WebGPU 的设备走 Cloud 额度
- 用户体感依然是"买断"，不是订阅
- 创作者 API 支出可控（仅兜底场景）

### 为什么不做月租
- 精神避难所的用户是"偶尔来"而非"每天用"
- 月租让人感觉自己"欠着服务"
- 买断符合"买一个安静角落"的心理模型
